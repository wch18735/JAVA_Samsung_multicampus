	■ 데이터베이스의 정의
 ⊙ 한 조직에서 여러 응용시스템들이 공용으로 사용하기 위해 통합 저장한 운영데이터의 집합
	
	■ 특징
 ⊙ 온라인 접근성
	- 질의에 대한 실시간 처리 및 응답
 ⊙ 계속적인 변화
	- 새로운 데이터의 삽입, 기존데이터의 삭제, 갱신으로 최신의 상태를 유지
 ⊙ 동시 공용
	- 서로 다른 목적을 가진 응용 사용자들이 동시에 사용

	■ 데이터 베이스 사용자
 ⊙ 응용 프로그래머
	- 일반사용자가 사용할 수 있도록 하는 프로그램을 작성해주는 프로그래머
	- java 등 프로그램 언어를 이용해서 DBMS를 사용
	
 ⊙ DBA (DataBase Admin: 데이터베이스 관리자)
	- DBMS 데이터베이스의 관리를 맡은 사람

	■ DBMS (Data Base Management System)
 ⊙응용프로그램과 데이터의 중재자로서 모든 응용 프로그램들이 데이터 베이스를 공유하도록 
   해주는 소프트웨어 시스템

 - 데이터 정의: 데이터의 구조를 정의하고 삭제 및 변경 기능을 수행
 - 데이터 유지보수: 데이터 삽입, 수정, 삭제를 관리
 - 데이터 추출: 사용자가 저장된 데이터를 검색하면 응용프로그램이 데이터를 추출
 - 데이터 제어: 데이터베이스 사용자를 생성하고 모니터링하고 접근을 제어

 ⊙ 정의기능
 - 하나의 저장구조로 여러 사용자 관점을 만족시킬 수 있도록 데이터 베이스 구조를 정의할 수 있는 기능
 
 ⊙ 조작기능
 - 사용자의 요구에 따라 체계적으로 데이터베이스를 접근하고 조작할 수 있는 기능
  > 데이터의 검색, 갱신, 삽입, 삭제 드 데이터베이스 연산 지원
 
 ⊙ 제어기능
 - 공용 목적으로 관리되는 데이터 베이스의 내용에 대해 데이터의 정확성과 안정성을 유지하는 기능

	■ SQL (Structured Query Language)
	: 관계형 데이터 베이스 관리시스템에서 데이터 베이스 스키마 생성, 자료의 검색, 관리, 수정, 데이터 베이스 객체 접근 관리 등을 위해 고안된 언어
	: 자바 응용프로그램에서도 SQL로 작성된 명령어를 DBMS에 보내서 데이터 베이스 처리를 지시함
	: 데이터 정의어 (DDL), 데이터 조작어(DML), 데이터 제어어(DCL) 기능 모두 제공
